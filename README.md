## unsafe.c 的问题
### 1. 竞态条件
当多个线程（如t1和t2）同时访问和修改同一个共享资源（全局变量`counter`）时，由于线程调度的不确定性，会导致执行结果依赖于线程执行的顺序，这种现象称为**竞态条件**

### 2. `counter++`的非原子性
`counter++`这行看似简单的语句，在底层并非原子操作（不可分割的最小单位）。它实际上对应三条机器指令[2,4](@ref)：
1.  **读取**：将`counter`的值从内存加载到CPU寄存器。
2.  **修改**：在寄存器中将值加1。
3.  **写入**：将新值写回`counter`的内存地址。
